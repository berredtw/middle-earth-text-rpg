# 魔戒 · 中土遠征 — 開發紀錄

純文字連線 RPG，世界觀依據托爾金《魔戒》，向 30 年前的 MUD 與《天堂》致敬。
開發期間：2026-07-06 ～ 2026-07-07（v1 ～ v13）。開發方式：使用者提出需求 → Claude Code 實作 → 自動化測試全數通過後交付。

---

## 版本沿革

### v1（2026-07-06）初版
- 單檔 `index.html` 遊戲本體：純文字、繁體中文、三欄版面（參考陸吼放置版：左＝角色狀態、中＝冒險與日誌、右＝分頁面板）
- 4 種族（人類遊俠／精靈／矮人／哈比人）各 4 技能、六項能力配點創角
- 主線 10 章（夏爾→末日火山）、14 張地圖、首領戰
- 天堂式強化卷軸（+3 安全／+4~6 六成／+7 起三成且失敗蒸發、祝福卷軸保底）
- 至尊魔戒腐化系統（戰鬥脫身 vs 腐化滿百戒靈懲罰）、放置自動戰鬥
- localStorage 3 存檔位＋匯出/匯入存檔碼（base64）
- `server.js` 零相依 Node WebSocket 伺服器（手刻 HTTP upgrade 交握與訊框解析）：通關密語（預設 mellon）、聊天、組隊（上限 4 人）、隊伍狩獵（隊長端模擬＋戰報轉發）、PvP 決鬥（挑戰者端模擬）

### 關鍵資料格式
- `MOBS[id] = [名, lv, hp, atk, def, exp, gold, drops[[id,%]], 半獸人?, 首領?, 屬性(10), 特性(11)]`（10/11 由 ELEM_MAP/MOB_FX 後掛）
- 裝備實例 `{id, e(強化), ench(附魔)}` — equipIt/unequip **必須攜帶 ench**
- 存檔 `S`：v2 格式；`S.opt` 放置設定、`S.tw` 塔進度、`S.comps/comp/compHp` 夥伴、`S.hasteUntil` 加速；舊檔用 `migrate()` 與 enterGame 內的補欄位守衛相容
- 戰鬥 `battle = {mobs[], tgt, buffs[], turn, floor, pPoison, pStun}`

### 平衡公式速查
- 經驗需求 `30×lv²`；玩家 maxHp `20+con×4+lv×8`（活力附魔 ×1.08）
- 塔怪 `lv=baseLv+floor×0.35`，hp `lv²×0.9`（樓主 ×2.2）、atk `lv×2.4`
- 強化：+3 內必成／+4~6 60%／+7 起 33% 失敗蒸發；武器每 +1＝攻+2、防具＝防+1
- 合成裝定位：略高於同級商店、低於首領/限定；最強序：商店 < 合成 < 首領(格蘭瑞37) < 塔頂(42/47) < 安都瑞爾44 < 外傳傳說(哈比66/上古76) < 格龍德80（魔苟斯稀有掉落，全遊戲最強）
- 品質詞綴（v13）：掉落/合成擲骰 60%普通/25%精良+10%/11%卓越+20%/4%傳奇+30%，套用攻防與售價

### 開發注意事項（教訓）
1. **塔怪勿用基礎怪倍率**縮放（高基數怪 70 層即爆表），一律用樓層等級公式。
2. 遊戲速度用 **100ms interval＋nextTickAt 時間戳**，不要用固定 interval 跳拍（無法整除）。
3. 改任何 ITEMS/MOBS/MAPS/QUESTS/RECIPES 後：跑 `tools/gen_codex.js` 更新圖鑑、跑兩套測試。
4. 測試用假 DOM 需要 `firstChild` getter（log 清理迴圈）；模擬戰鬥要「死亡後移回原地圖」與「50% 血補滿」模擬喝藥。
5. BAT 檔規則：UTF-8 無 BOM、寫完跑 `python C:\Users\berred\fix_crlf.py`；PS1 禁跑 fix_crlf。
6. 連線協定：伺服器只做轉發與名單，戰鬥模擬都在客戶端（決鬥＝挑戰者端、狩獵＝隊長端），雙方顯示同一份戰報。
