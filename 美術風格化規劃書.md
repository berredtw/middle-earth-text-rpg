# 魔戒 · 中土遠征 — 美術風格化規劃書（v2 手繪插畫版）

給 Gemini（或任何 AI 繪圖工具）畫圖用的完整規格文件。目標：把現在「純文字＋方框」的介面，改造成**橫向卷軸手繪插畫風格**——城鎮有背景與可點選的實體建築，每張地圖有專屬場景，每隻怪物、四大種族主角、招募夥伴都有像素圖示。

## 1. 現況與改造範圍

目前 `index.html`（純前端單檔）完全沒有任何圖片素材，所有畫面都是 CSS 方框＋文字：

- **戰鬥畫面**（`renderBattle()`）：敵人是一格一格的 `.mobcard` 文字方塊（名稱／等級／血條），沒有場景、沒有立繪。
- **城鎮畫面**（`renderLoc()`）：城鎮只是文字說明＋一排按鈕（🛒商店／🛏旅店／🍺招募／🔨工房／🧙甘道夫／📜懸賞板），沒有任何畫面。
- **地圖（野外）畫面**：同樣只有文字說明＋「🔍探索」「☠挑戰首領」按鈕，沒有背景圖。
- **種族／夥伴**：創角與夥伴名冊都只有文字描述，沒有頭像。

本次規劃要新增的視覺層，全部是**疊加在現有版面上的圖片**，不需要重寫遊戲邏輯，所以可以先把圖畫好，之後再一次接進去。

## 2. 總體美術方向

### 2.1 風格參考

- **手繪插畫風**（Hand-painted Illustration），比照高品質 JRPG 主視覺／設定集的質感：線條乾淨、上色厚實、輪廓清楚。
- **絕對不要像素風／馬賽克**——Gemini 產的是「假像素」（實際上是高解析圖疊馬賽克效果），事後程式清不掉，一律改用平滑手繪。
- 不要 3D 算圖、不要真實照片質感、不要現代扁平插畫風。
- 整體基調：托爾金式史詩奇幻，偏暗沉、厚重、帶點油畫感的中世紀氣氛（不要卡通可愛風，也不要賽博龐克霓虹風）。

### 2.2 調色盤（沿用網頁現有主題色，讓圖片跟介面融合）

畫面／UI 目前用這組色票，請 Gemini 產圖時盡量讓場景的暗部、金屬邊框呼應這些顏色：

| 用途 | 色碼 | 說明 |
|---|---|---|
| 背景底色 | `#0d0f12` | 近黑，畫面最暗處可對齊 |
| 面板色 | `#16191f` / `#1c2028` | 深灰藍，UI 框底色 |
| 邊框線 | `#2e3542` | 冷灰藍細邊 |
| 主題金色 | `#d4a94e` | 標題、強調邊框、按鈕 hover，**建議所有 UI 外框／圖示的金屬邊都用這個金色** |
| 次要文字 | `#8a93a5` | 灰藍 |
| 危險紅 | `#e05252` | 敵方／傷害 |
| 藍 | `#5b8dd9` | |
| 綠 | `#5fbf7f` | 治療／安全 |
| 紫 | `#a77bd9` | 特殊狀態 |

場景背景本身可以更豐富（森林綠、雪地白、熔岩橘紅……），但**建築物／UI 相關的金屬裝飾統一用金色 `#d4a94e`**，怪物/夥伴卡片外框沿用暗紅 `#5a3030`（現有 `.mobcard` 邊框色），維持一致性。

### 2.3 視角與構圖

- **場景背景**（城鎮、野外地圖）：橫幅構圖、視角略俯（3/4 俯視或平視皆可，但整批風格要統一），近似「舞台佈景」，因為畫面上還要疊放血條、按鈕等 UI，**背景下半部盡量留深色/簡單區域**，避免與文字衝突。
- **角色／怪物立繪**：正面或 3/4 側面全身像，站姿為主（見第 4、8 節動作規格），**去背透明 PNG**。
- **建築物（城鎮熱點）**：3/4 側視小型建築插畫，類似「城鎮地圖上的一棟房子」，**去背透明 PNG**，之後會疊在城鎮背景上。

## 3. 檔案規格與命名規則

請 Gemini 畫完後依照下表存檔，方便之後直接放進專案資料夾對號入座。

```
assets/
  ui/                補檔（邊框、血條、按鈕圖示、屬性圖示）
  races/              四大種族主角立繪
  companions/          夥伴頭像
  mobs/                怪物立繪（家族基礎款＋首領原創款）
  towns/               城鎮背景（含建築熱點，见第5節）
  biomes/              野外／地城場景背景
```

| 類別 | 建議畫布尺寸 | 格式 | 背景 |
|---|---|---|---|
| 場景背景（城鎮／野外／地城） | 1280×720（16:9，橫向卷軸感） | PNG | 不透明，滿版 |
| 城鎮建築熱點（單棟） | 200×220 上下（依比例，直式） | PNG | **透明背景去背** |
| 種族主角立繪 | 256×384（直式全身） | PNG | **透明背景去背** |
| 夥伴頭像 | 192×192（半身或全身） | PNG | **透明背景去背** |
| 怪物立繪－一般款 | 160×160 | PNG | **透明背景去背** |
| 怪物立繪－首領款 | 320×320（體型較大，畫面感要足） | PNG | **透明背景去背** |
| UI 圖示（技能／屬性／按鈕） | 64×64 | PNG | **透明背景去背** |

命名請直接用遊戲資料裡的英文代號（見第 6、8、9 節對照表），例如 `mobs/wolf.png`、`towns/t_hobbiton.png`、`races/human.png`——這樣我接圖時可以直接寫程式對應，不用一張張手動改名。

**風格提醒（v2）**：所有 prompt 結尾都要帶一句「absolutely NOT pixel art: no pixelation, no mosaic, no visible pixel grid, no dithering」。Gemini 很愛自作主張加像素濾鏡，務必明講禁止；產出若仍帶馬賽克直接重抽，不要試圖後製修復。

## 4. 戰鬥畫面改造：橫向卷軸場景

### 4.1 新版面概念

現在戰鬥區塊（`#loc-body` 內、`renderBattle()` 產生的內容）要從「文字方塊列表」改成一個**分層舞台**：

```
┌───────────────────────────────────────────────────────────┐
│  （最上層）行動按鈕列：⚔攻擊／技能／🧪藥水／🏃逃跑           │
├───────────────────────────────────────────────────────────┤
│                                                             │
│   [場景背景：依所在地圖套用對應 Biome 背景圖]                 │
│                                                             │
│   🧑主角立繪        🐺 🐺 🐺  （敵人沿地平線橫向排開）         │
│   👤夥伴立繪(若有)                     ▲ 目前選中目標有金框    │
│   （站在畫面左側地面）                                        │
│                                                             │
├───────────────────────────────────────────────────────────┤
│  每隻怪物下方疊一條手繪風血條 + 名稱牌（沿用現有資料，圖形化）  │
└───────────────────────────────────────────────────────────┘
```

- **背景層**：整場戰鬥用「目前所在地圖」對應的 Biome 背景圖（見第 6 節），滿版鋪底，`background-size:cover`。
- **主角層**：玩家種族立繪固定站在畫面左側偏下（地平線上），夥伴立繪疊在主角旁（稍小、稍後）。
- **敵人層**：怪物卡片改成「立繪＋腳下血條」，沿著背景的地平線橫向一字排開（1～5 隻皆可，用 flex 由左至右排列，取代現在的 grid 方塊）；被選中的目標腳下加一個金色箭頭／光圈標記（取代現在的金色邊框）。
- 「橫向卷軸式」指的是整個戰鬥場景做成**寬幅橫圖**（1280×720 或更寬的 21:9），角色與怪物像站在一條舞台上，而不是現在的「每隻怪物一個獨立小方塊」。不需要真的做卷動（scroll）功能，用固定寬幅構圖＋左右排列就能有卷軸感；如果想更講究，背景可以做「近景／遠景」兩層（例如遠山不動、近處草叢/戰場殘骸）做出簡單視差，但這是加分項目，非必要。

### 4.2 立繪動作需求（給 Gemini 的實際產出量）

因為 Gemini／Nano Banana 這類工具**不擅長畫連續動畫幀**（畫出來的每張很難完全一致，做成流暢動畫很吃力），建議**每個角色只需要 1 張靜態站姿立繪**，之後攻擊/受擊的動態效果用程式碼做簡單效果（例如 CSS：攻擊時整張圖往前頓一下、受擊時閃白／震動、死亡時淡出倒下），不需要 Gemini 畫出「攻擊動作」「揮劍瞬間」等額外幀。

如果想要更豐富，可以額外加一張「攻擊姿勢」（例如舉劍／拉弓瞬間）作為選配，但**站姿一張是必需、優先**。

## 5. 城鎮畫面改造：背景＋可點建築

### 5.1 現況對照：文字按鈕 → 實體建築熱點

目前城鎮（`renderLoc()` 裡 `m.town` 分支）是一排文字按鈕，要改成「背景圖＋建築物疊圖，滑鼠點建築＝原本按鈕功能」：

| 原本文字按鈕 | 對應函式 | 改造成的建築物件 |
|---|---|---|
| 🛒 商店 | `openShop()` | 市集攤位／雜貨鋪門面（掛著武器防具招牌） |
| 🛏 旅店休息 ／ 🍺 旅店招募夥伴 | `restInn()` / `recruit()` | 旅店建築（同一棟樓，掛酒桶／床鋪招牌；兩個按鈕可疊在同一個熱點上，點擊後彈出「休息」或「招募」兩個子選項） |
| 🔨 工房合成 | `openCraft()` | 鐵匠鋪（風箱＋鐵砧＋煙囪冒煙） |
| 🧙 甘道夫的附魔 | `openGandalf()` | 巫師的篷車／小屋（掛著星星圖騰、煙斗煙霧） |
| 📜 懸賞板 | `openBounty()` | 城鎮公佈欄／告示牌（插滿羊皮紙） |
| ⏳ 新的輪迴 ／ 🌋 投戒 ／ ⚔ 傳說迴廊 | 特殊劇情限定 | 只在特定城鎮出現的地標物件（見下） |

### 5.2 每個城鎮的「地標建築」（背景主視覺）

除了上述 5 個**共通功能熱點**（每個城鎮都要有），每張城鎮背景還要有一個**專屬地標**，讓 10 個城鎮長得不一樣、有辨識度：

| 城鎮代號 | 城鎮名 | 地標建築／場景重點 |
|---|---|---|
| `t_hobbiton` | 夏爾．哈比屯 | 圓形哈比人地洞門排成一列、山丘上的派對樹 |
| `t_bree` | 布理．躍馬旅店 | 躍馬旅店本體（招牌畫著一匹躍起的馬） |
| `t_rivendell` | 瑞文戴爾．愛隆之家 | 幽谷瀑布、精靈風格白色迴廊建築 |
| `t_lorien` | 羅斯洛立安 | 巨大瑁瓏樹上的樹屋、金色燈火 |
| `t_edoras` | 伊多拉斯．黃金宮殿 | 山丘上金頂梅杜賽德大殿 |
| `t_minas` | 米那斯提力斯．白城 | 七層白色石城、頂端聖白樹 |
| `t_gate` | 時光之門．無時之廳 | 星光織成的巨大門扉、白色大廳 |
| `t_laketown` | 長湖鎮．伊斯加羅 | 湖面木樁上的水上木造市鎮 |
| `t_valinor` | 維林諾．雙樹之地 | 銀樹泰爾佩瑞安＋金樹羅瑞林並立 |
| `t_havens` | 灰港岸．米斯瀧德 | 白色港口碼頭、一艘停泊的白船 |

### 5.3 建議做法

有兩種實作方式，**建議 Gemini 直接畫「方式 A」**（比較省事、畫面也比較自然）：

- **方式 A（推薦）**：每個城鎮畫**一張完整背景圖**（1280×720），把地標建築＋5 個功能建築都直接畫進同一張構圖裡（類似遊戲美術的一張城鎮全景圖），不用去背。之後我用 CSS 在對應座標上疊「透明可點擊區域」（不需要額外圖層，只要告訴我每個建築大概在畫面的哪個位置，例如「旅店在左側 20% 處」）。
- **方式 B（較彈性但工作量大）**：背景畫「空城」（只有地面、天空、地標），5 個功能建築各畫一張獨立透明去背小圖，之後用程式碼把建築物「擺」到背景上，好處是同一批建築圖可以在不同城鎮背景重複使用、單獨替換。

若採方式 A，請在畫完後**簡單標註**（用文字說明即可，不用真的畫格線）：這張城鎮背景裡，商店／旅店／工房／甘道夫／佈告欄，大概各自在畫面的什麼位置（例如「由左至右：旅店、市集、鐵匠鋪、佈告欄、巫師篷車」），方便我之後對應點擊熱區座標。

## 6. 地圖背景清單（場景／Biome）

野外地圖加上地城塔總共約 **50 張**，如果每張都畫一張全新原創背景，對單人開發＋AI 產圖是不小的工作量。建議**歸類成約 20 種「場景模板」（Biome）**，同類地圖共用同一張背景（可請 Gemini 用同一張圖做小幅度色調變化，比如同一片森林在「白天」「黃昏」「戰火中」三種色調），大幅降低要畫的張數，畫面上也不會有違和感（原著中很多地圖本來就是同一種地形的不同路段）。

## 12. Gemini 提示詞（Prompt）範本合集

以下範本可直接複製到 Gemini，把 `【　】` 內的文字換成對應的名稱／描述即可。每個範本都已經包含風格關鍵字，確保產出風格一致。

### 12.1 城鎮背景範本

```
Wide hand-painted fantasy illustration, fantasy medieval town, 16:9 landscape composition, 1280x720,
clean crisp linework, rich harmonious colors, sharp focus, the style of high-quality classic JRPG key art,
dark epic Tolkien-inspired fantasy mood, warm torchlight accents, gold-trimmed architecture details (#d4a94e accent color).

Scene: 【城鎮名稱，例如：夏爾的哈比屯，圓形地洞門排成一列，山丘上有一棵大樹】。
The composition should clearly include, arranged left to right across the scene:
an inn/tavern building with a hanging sign, a marketplace stall, a blacksmith's workshop with a chimney,
a notice board covered in scrolls, and a wizard's wagon or small tower with a starry banner.
Lower third of the image should be relatively open ground/path so UI elements can be overlaid later.
No characters in the scene, empty street.
```

### 12.8 產圖一致性小技巧

- 先畫 Phase 0 的第一張城鎮背景，**滿意之後把這張圖回傳給 Gemini 當作「風格參考圖」**（image-to-image 或在 prompt 附上「keep the same art style as the reference image」），後續每一批新圖都附上同一張參考圖，可以大幅提高幾十張圖之間的風格一致性。
- 同一批（例如所有怪物家族基礎款）盡量在同一個對話串裡連續生成，減少 Gemini 重新「抓風格」的落差。
- 若產出帶有馬賽克／像素格，直接重新生成並強調 STYLE RULE，不要嘗試後製移除（假像素混在圖裡清不乾淨）。
